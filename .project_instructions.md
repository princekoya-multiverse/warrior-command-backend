# Warrior Command Center Backend

## Overview
Institutional-grade Web3-native operations HQ backend API built with NestJS + TypeScript + Prisma + PostgreSQL.

## Architecture
- **Framework**: NestJS (modular enterprise architecture)
- **Database**: PostgreSQL with Prisma ORM
- **Vector Store**: Qdrant (optional, HTTP API integration - gracefully degrades when unavailable)
- **LLM**: Abacus.AI API for session summarization and embeddings
- **Real-time**: WebSocket (Socket.IO) gateway at /ws namespace
- **LiveKit**: Integration endpoints (placeholder until LiveKit is deployed on Akash)

## Modules
- **UsersModule**: User CRUD + API key generation (bcrypt hashed)
- **ProjectsModule**: Project CRUD with owner/status filtering
- **EpicsModule**: Epic CRUD with nested hierarchy support
- **TasksModule**: Task/subtask CRUD with today/week views, status/owner filtering
- **DailyLogsModule**: Daily log entries with mood/energy tracking
- **AgentsModule**: AI agent registration and management
- **SessionsModule**: Session management with full handoff protocol
- **MemoryModule**: Persistent agent memory with semantic search
- **GatewayModule**: WebSocket events (task.*, session.*, agent.*)
- **LivekitModule**: LiveKit room/token management
- **LlmModule**: Abacus.AI integration for summaries/embeddings
- **QdrantModule**: Vector store via HTTP API

## Session Handoff Protocol
1. Close session → generate LLM summary → extract decisions/actions
2. Store summary + embeddings in Qdrant + PostgreSQL
3. Create session_handoff record linking sessions
4. New session loads context from handoff + semantic search

## Database Schema
10 models: user, project, epic, task, daily_log, agent, session, message, agent_memory, session_handoff

## Key Design Decisions
- API key auth via x-api-key header (bcrypt hashed in DB)
- Qdrant uses HTTP API directly (avoids ESM-only @qdrant/js-client-rest)
- UUID generation via crypto.randomUUID()
- All external services (Qdrant, LiveKit) gracefully degrade when not configured
- Swagger docs at /api-docs with custom styling

## Deployment
- Dockerfile: multi-stage Node 20 Alpine build
- Health check at /health
- Port 3000
- Designed for Akash Network deployment
